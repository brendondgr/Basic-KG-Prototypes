Graph Fundamentals (Nodes & Relationships)

Neo4j is a native graph database that represents data using the Property Graph Model. The data structures are composed of:

• Nodes: Entities (e.g. Person, City) containing key-value property structures.
• Relationships: Directed, named connections connecting nodes directly.

Every relationship has a direction, a single type label (e.g. LIVES_IN), and can hold properties just like nodes.

Pattern Matching Query

Cypher uses visual ASCII art patterns to represent relationships. This query matches residents in a city where a company is headquartered:

Cypher
MATCH (p:Person)-[:LIVES_IN]->(c:City) MATCH (com:Company)-[:HQ_IN]->(c) RETURN p.name, com.name;

Graph Fundamentals (Vertices & Edges)

Gremlin is an **imperative and declarative graph traversal language** developed by Apache TinkerPop. It traverses property graphs consisting of:

• Vertices: Equivalent to nodes, represent objects/entities.
• Edges: Equivalent to relationships, represent directed connections.

While the models are identical conceptually, Gremlin code explicitly references steps over vertices (V()) and edges (E()).

Step-by-step Traversal Query

Gremlin queries are pipelines of map/filter operations. This traversal starts at all Person vertices and steps outward along LIVES_IN edges:

Gremlin
g.V().hasLabel("Person").out("LIVES_IN").values("name")
