Knowledge Graph Tutorial
01

Graph Fundamentals

Unlike tabular data, Knowledge Graphs represent information as entities (nodes) and relationships (edges). This allows for complex, recursive querying.

Every piece of data is stored as a Triple:
(Subject) — [Predicate] — (Object)

02

Ingestion Pipeline

To populate a graph from a relational source, we use a Schema-First approach to map vertices and edges.

LOAD csv.file("data.csv") AS line MERGE (p:Person {id: line.id}) CREATE (p)-[:LIVES_IN]->(:City {name: line.city});
03

Querying with Cypher

The power of KGs lies in Pattern Matching. Find all people who live in a city where a specific company is headquartered:

MATCH (p:Person)-[:LIVES_IN]->(c:City) MATCH (com:Company)-[:HQ_IN]->(c) RETURN p.name, com.name;
